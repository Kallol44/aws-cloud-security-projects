# AWS Security Monitoring & Alerting Project

This project demonstrates a hands-on approach to setting up AWS native security monitoring and real-time alerting using core services such as GuardDuty, CloudTrail, AWS Config, and EventBridge.

## ✅ Project Phases

### 📘 Phase 0: Prerequisites
- AWS Free Tier Account
- AWS CLI installed and configured
- Budget alarm configured

### 🔐 Phase 1: Enable Core AWS Security Services
- CloudTrail, GuardDuty, AWS Config, CloudWatch Logs, and SNS

### 🧪 Phase 2: Simulated Attack – Public S3 Bucket Access
- Simulates public access to S3 bucket and triggers GuardDuty alerts

### 🚨 Phase 3: Create Real-Time Alerts
- Uses EventBridge to send alerts from GuardDuty to SNS email
